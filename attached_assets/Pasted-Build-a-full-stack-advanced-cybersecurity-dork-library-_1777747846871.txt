Build a full-stack advanced cybersecurity dork library website.

Project goal:
Create a visually modern, interactive website that collects, categorizes, stores, searches, and displays Google dork-style cybersecurity search patterns from multiple public sources such as GitHub, Medium RSS, security blogs, RSS feeds, and manual admin submissions.

Important:
Do not include real dangerous targets or real leaked secrets. Use placeholders like example.com, {domain}, {company}, {keyword}, {filetype}, {technology}. The app should support cybersecurity learning, OSINT organization, exposure monitoring, and authorized research workflows.

AI processing pipeline:
Create services/functions for:

1. fetchSources()
- Fetch latest content from configured sources.
- GitHub: search public repositories and markdown files based on keywords.
- RSS: parse articles.
- Manual: allow admin to paste raw text or URLs.
- Store all fetched text in RawContent.
- Deduplicate using contentHash.

2. extractDorkCandidates(rawText)
- Detect dork-like search patterns.
- Look for common search operator structures such as site:, filetype:, intitle:, inurl:, intext:, ext:, cache:, related:.
- Extract candidate queries.
- Store in ExtractedDork.

3. normalizeDork(rawQuery)
- Trim spaces.
- Standardize operator casing.
- Replace real domains with placeholders where appropriate.
- Remove duplicate operators.
- Convert into queryTemplate.

4. categorizeDork(normalizedQuery, context)
- Use AI to classify into category, subcategory, intentType, difficulty, tags, platforms, and operators.

5. optimizeDork(normalizedQuery, category)
- Create a cleaner and more readable version.
- Do not generate real targets.
- Use placeholders.
- Add useful metadata.
- Generate title and explanation.
- Suggest related dorks.

6. importExtractedDork(extractedDorkId)
- Convert ExtractedDork into Dork.
- Create DorkVersion.
- Link categories, tags, operators, and platforms.
