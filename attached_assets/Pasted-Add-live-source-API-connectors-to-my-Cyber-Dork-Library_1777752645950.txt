Add live source API connectors to my Cyber Dork Library project.

Important safety rules:
- Do not store real domains, real companies, real emails, real API keys, real tokens, real passwords, leaked secrets, or exploit instructions.
- Any query pattern imported from live sources must be normalized into safe placeholders such as {domain}, {company}, {keyword}, {technology}, {filetype}, {cloud_provider}, {platform}, {date}.
- All live results must go first into RawContent and ExtractedDork review queue.
- Do not auto-publish imported dorks unless an admin reviews/imports them.

Add these environment variables to .env.example:

GOOGLE_CSE_API_KEY=
GOOGLE_CSE_ID=
GITHUB_TOKEN=
RSS_FEEDS=
MEDIUM_FEEDS=
AI_API_KEY=
AI_MODEL=
AI_BASE_URL=

Create these files:

1. src/lib/live-sources/google-cse.ts
- Use Google Custom Search JSON API.
- Function: searchGoogleDorkPatterns(query: string)
- Call:
  https://www.googleapis.com/customsearch/v1?key=${GOOGLE_CSE_API_KEY}&cx=${GOOGLE_CSE_ID}&q=${encodeURIComponent(query)}
- Return title, snippet, link, sourceType: "GOOGLE_CSE".
- Store only metadata/snippet in RawContent.
- Never store dangerous real target queries directly.
- Run sanitizer before storing.

2. src/lib/live-sources/github.ts
- Use GitHub REST API.
- Function: searchGitHubDorkPatterns(query: string)
- Search public repositories and code/markdown files for educational dork pattern content.
- Use Authorization: Bearer ${GITHUB_TOKEN}.
- Suggested safe search terms:
  "google dork examples cybersecurity"
  "google dork cheatsheet filetype md"
  "osint dork examples"
  "security dork cheat sheet"
  "defensive google dork monitoring"
- Fetch README/markdown content only when rate limits allow.
- Store source URL, title, and safe excerpt in RawContent.
- Sanitize before storing.

3. src/lib/live-sources/rss.ts
- Use rss-parser package.
- Function: fetchRssFeeds()
- Read RSS_FEEDS and MEDIUM_FEEDS from env as comma-separated URLs.
- Fetch title, link, contentSnippet/content.
- Store as RawContent.
- Sanitize before storing.

4. src/lib/safety/sanitize-dork.ts
Create:
- sanitizeDorkText(input: string): string
- normalizeDorkQuery(input: string): string
- containsDangerousSecret(input: string): boolean

Sanitizer rules:
- Replace domains with {domain}.
- Replace company-like names with {company} when used as targets.
- Replace emails with {email}.
- Replace IP addresses with {ip_address}.
- Replace API keys, tokens, bearer tokens, private keys, password-like strings with {secret_placeholder}.
- Replace filenames that look sensitive with {filetype} or {keyword}.
- Keep only educational query templates.
- Remove or reject content that contains real leaked secrets, credential dumps, exploit steps, or instructions to abuse systems.
